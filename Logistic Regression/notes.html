Why Not Linear Regression for Classification?

Your notes used student pass/fail (Pass=1, Fail=0) as the example:

Linear Regression gives: 1.4 ← what does that even mean? Or: −0.3 ← negative probability, impossible!

Linear Regression predicts values below 0 and above 1. Probability must stay between 0 and 1 — so linear regression fails here.

Logistic Regression Fixes This

Always outputs between 0 and 1

Instead of predicting a value directly, it predicts probability.

Sigmoid is a probability machine. It converts any number into a probability between 0 and 1.

Linear Regression

Input → Number (can be anything)

Logistic Regression

Input → Probability (always 0–1)

Threshold — Making the Decision

Default threshold = 0.5 Probability > 0.5 → Yes (1) Probability < 0.5 → No (0)

Example: P = 0.80 → 80% chance Yes. P = 0.20 → 20% chance No.

Coefficients — Loan Approval Example

Income coefficient = +2.5 → increases approval chance Debt coefficient = −3.0 → decreases approval chance

Coefficients represent the change in log-odds for a one-unit increase in the predictor.

The Sigmoid Function

h_θ(x) = 1 / (1 + e^(−z))

It "squashes" any real number into a value between 0 and 1.

g(z) > 0.5 when z ≥ 0 → predict class 1. g(z) < 0.5 when z < 0 → predict class 0.

Non-Convex → Convex via Sigmoid

Your notes drew two curves — this is crucial. The raw cost surface for logistic regression is non-convex (many valleys), which would cause gradient descent to get stuck in local minima.

Applying the sigmoid + log loss transforms it into a convex function with a single global minimum — gradient descent always finds it.

Why Not MSE for Logistic Regression?

Linear Regression uses MSE. Logistic Regression uses Log Loss (Cross Entropy Loss).

Probabilities behave differently than numerical predictions. Wrong predictions should be punished heavily.

The Cost Function

Two cases from your notes:

If y = 1: Cost = −log(h_θ(x)) If y = 0: Cost = −log(1 − h_θ(x))

Combined into one formula (the one you boxed in red):

Cost = −y·log(h_θ(xⁱ)) − (1−y)·log(1 − h_θ(xⁱ))

When y=1, the second term vanishes. When y=0, the first term vanishes. Both cases collapse into one elegant formula.

Intuition — Why Log?

y=1, predict 1

−log(1) = 0 ✓
No penalty

y=1, predict 0

−log(0) = ∞ ✗
Huge penalty

Being confident and wrong is punished extremely hard.

Gradient Descent for Logistic Regression

1
Find error — compare prediction to actual
2
Reduce error — compute gradient of log loss
3
Update coefficients — step in direction of lower cost
4
Repeat until convergence

Same three ingredients as linear regression: Gradient Descent, Learning Rate, Convergence.

Accuracy

Accuracy = (TP + TN) / (TP + FP + FN + TN)

Your example: (3+1)/(3+2+1+1) = 4/7 = 57%

Problem with accuracy: If 0→900 and 1→100 (imbalanced), a model predicting all 0s scores 90% accuracy but is completely useless.

For imbalanced data → use Precision, Recall, F-score instead.

Precision vs Recall

Precision

TP / (TP + FP)

Trustworthiness — of all things I predicted positive, how many actually are?

FP is the enemy

Recall

TP / (TP + FN)

Detection power — of all actual positives, how many did I catch?

FN is the enemy

The Trade-off

Increasing recall often decreases precision and vice versa. That's why we need a way to balance them → F-Score.

F-Score Formula

F_β = (1 + β²) × (Precision × Recall) / (β² × Precision + Recall)

β controls which you care about more: FP or FN.

Three Common F-Scores

ScoreβFocusWhen to use
F0.50.5FP >> FNPrecision matters more (e.g. spam filter)
F11FP = FNBoth equally important
F22FN >> FPRecall matters more (e.g. cancer detection)
F1 = 2 × (Precision × Recall) / (Precision + Recall)

The β Rule from Your Notes

Decrease β (e.g. 0.5)

More importance to false positives → care more about Precision

Increase β (e.g. 2)

More importance to false negatives → care more about Recall

Assumptions of Logistic Regression

1
Binary outcomes — 2 output classes
2
Independent observations — each row should be unrelated to others
3
No multicollinearity — features should not influence each other
4
Large sample size preferred — probability estimates are unstable on small datasets
5
Linear relationship between predictors and log-odds (not probability directly)
Linearity in Logistic Regression = predictors have a linear relationship with log-odds, not with probability itself.

# Linear Regression with 5-fold Cross-Validation
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import cross_val_score
import numpy as np

lin_reg = LinearRegression()
mse = cross_val_score(
    lin_reg, X, y,
    scoring='neg_mean_squared_error',
    cv=5    # 5-fold → 5 different MSE values
)

Logistic Regression

from sklearn.linear_model import LogisticRegression
from sklearn.datasets import load_breast_cancer
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report

# Load data
data = load_breast_cancer()
X, y = data.data, data.target
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Train
model = LogisticRegression()
model.fit(X_train, y_train)

# Predict
y_pred  = model.predict(X_test)
y_proba = model.predict_proba(X_test)

# Evaluate (shows precision, recall, F1)
print(classification_report(y_test, y_pred))
