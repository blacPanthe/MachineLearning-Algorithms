KNN Has No "Model" — It Just Remembers

Linear Regression learns weights. Logistic Regression learns a sigmoid curve. KNN learns nothing in advance — it just stores the entire training set.

KNN is called a "lazy learner": all the work happens at prediction time, not training time.

How a Prediction Is Made

1
A new point arrives that needs a prediction.
2
Measure the distance from this point to every point in the training set.
3
Pick the K closest training points (the "neighbors").
4
Vote: the new point gets the majority class among those K neighbors.

Euclidean Distance (default)

d(p, q) = √[ (p₁−q₁)² + (p₂−q₂)² + ... + (pₙ−qₙ)² ]

The straight-line distance between two points across all feature dimensions. This is what sklearn's KNeighborsClassifier uses by default.

Manhattan Distance

d(p, q) = |p₁−q₁| + |p₂−q₂| + ... + |pₙ−qₙ|

Sums absolute differences instead of squaring them — like navigating a city grid instead of cutting diagonally. Less sensitive to outliers in any single feature.

Minkowski Distance

A general formula that includes both of the above as special cases:

p = 1 → Manhattan p = 2 → Euclidean

sklearn's p parameter on KNeighborsClassifier controls this directly.

The K Trade-off

Small K (e.g. K=1)

Overfits Very sensitive to noise — a single mislabeled neighbor flips the prediction.

Large K (e.g. K=100)

Underfits Smooths out local patterns — minority classes get drowned out by the majority vote.

How to Pick K in Practice

Try a range of K values (e.g. 1 to 30), measure accuracy (or a better metric) on a validation/test set for each, and pick whichever K performs best.

There's no formula for the "right" K — it's found empirically, by testing.

A Useful Rule of Thumb

Use an odd K for binary classification, so a tie vote (e.g. 2 vs 2) can't happen.

Distance Is the Whole Model — So Scale Matters MORE Here

In Linear/Logistic Regression, scaling makes coefficients comparable, but the model can still fit correctly without it. In KNN, scaling affects the prediction itself.

If one feature ranges 0-1000 and another ranges 0-1, the big feature alone decides who's a "neighbor" — the small feature is effectively ignored.

Example

Comparing wines by "total sulfur dioxide" (0-300) and "chlorides" (0.01-0.6) without scaling:

distance ≈ |Δ sulfur dioxide| (chlorides difference is mathematically negligible)

After StandardScaler, both features contribute roughly equally to the distance calculation.

DisadvantageWhy

The Curse of Dimensionality

As you add more features (dimensions), distances between points become less meaningful — everything starts looking roughly equidistant from everything else.

In very high dimensions, "nearest" neighbor stops being a useful concept — there's no longer a clear "near" and "far."

Why This Happens

The volume of space grows exponentially with each added dimension, so data points spread out and the gaps between them become more uniform. With only a handful of features (like the 11 wine measurements here), this isn't a big problem — but with hundreds of features, KNN's distance calculation can break down.

Mitigations

1
Feature selection — drop irrelevant features before computing distance
2
Dimensionality reduction — e.g. PCA, to compress features first
3
More data — higher dimensions need exponentially more points to stay "dense"

Basic KNN Classifier

# K Nearest Neighbour classifier
from sklearn.neighbors import KNeighborsClassifier
from sklearn.preprocessing import StandardScaler

# Scaling is essential — KNN is distance-based
scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test  = scaler.transform(X_test)

model = KNeighborsClassifier(n_neighbors=5)
model.fit(X_train, y_train)

y_pred  = model.predict(X_test)
y_proba = model.predict_proba(X_test)

Finding the Best K

import numpy as np

accuracies = []
for k in range(1, 31):
    knn_k = KNeighborsClassifier(n_neighbors=k)
    knn_k.fit(X_train, y_train)
    accuracies.append(knn_k.score(X_test, y_test))

best_k = range(1, 31)[np.argmax(accuracies)]
print("Best K:", best_k)

Choosing a Distance Metric

# p=1 -> Manhattan, p=2 -> Euclidean (default)
model = KNeighborsClassifier(n_neighbors=5, p=1)
