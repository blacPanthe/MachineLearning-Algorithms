What is Regression?

Used to predict continuous numerical values. When your target is a number (like salary, price, score), Linear Regression becomes the candidate.

5 Things Linear Regression Does

1
Find relationship → X → Y (input to output)
2
Create a line → y = mx + c
3
Measure error → How wrong are predictions?
4
Reduce error → Use optimization
5
Evaluate performance → R², RMSE, MAE, MSE

The Formula — Salary Example

Predicted value = starting value + (change per unit) × input

Salary = 30,000 + 5,000 × (years experience)

Employee with 4 years experience:

= 30,000 + 5,000 × 4 = 50,000

intercept = 30,000 (base salary) slope = 5,000 (per year)

Residual (Error)

Actual = 100, Predicted = 90

Residual = Actual − Predicted = 100 − 90 = 10
Regression is basically trying to make these errors as small as possible.

The coefficient = expected change in target for one unit increase in predictor, holding all others constant.

Why Not Just Draw Any Line?

Different lines produce different errors. We need a systematic way to measure which line is best → that's the Cost Function.

Cost Function

A mathematical function that measures how wrong a model's predictions are. For linear regression, the most common is:

MSE — Mean Squared Error

Why Square the Errors?

Imagine errors of +10 and −10:

Sum = +10 + (−10) = 0 ← Misleading!

Square them:

10² + 10² = 100 + 100 = 200 ← Honest!

Squaring also punishes large errors more than small ones.

MSE — "How Bad Is My Model?"

MSE = average of all (actual − predicted)²

Lower is better

Convergence

When additional iterations no longer significantly reduce error → training stops.

500 → 200 → 80 → 20 → 5 → 4.9 → 4.89 → 4.889 ← converging

For linear regression only one minimum exists — no local minima trap!

LINE Assumptions

L
Linear relationship — between predictors & target
I
Independence — observations should not be correlated with each other
N
Normal residuals — approximately normally distributed around zero
E
Equal variance (Homoscedasticity) — constant variance across all predictor levels

Plus: No multicollinearity

Multicollinearity ⚠️

Predictors should not be highly correlated with each other. Using age + years of experience together creates problems.

Symptoms

Weird coefficients
Large P-values
High standard errors

Solutions

Remove a variable
Feature selection
VIF analysis

Performance Metrics

MAE

Average absolute error. Easy to explain. Average prediction mistake.

MSE

Average squared error. Punishes large mistakes more heavily.

RMSE

√MSE. Most practical — same unit as target variable.

R²

% of variance explained by model. Higher is better.

RMSE > MAE > MSE (order of preference for practical use)

R² — "How Much Did My Model Understand?"

R² = 0.80 → 80% variance explained, 20% unexplained
R² ValueMeaning
0Useless
0.5Moderate
0.8Strong
1.0Perfect

Problem: Adding ANY variable (even useless ones) always increases R²!

Adjusted R²

Punishes unnecessary predictors. If a new variable is useless → Adjusted R² decreases.

Adjusted R² accounts for model complexity whereas R² does not.

When NOT to Use Linear Regression

Target is categorical Highly non-linear Severe outliers Strong multicollinearity

Why Regularization?

When there are many predictors or multicollinearity, linear regression may overfit. Ridge & Lasso add a penalty to the cost function.

Ridge Regression (L2)

Prevents overfitting by shrinking the slope. Never zeroes out coefficients.

Cost = (ŷ − y)² + λ(slope)²

λ is a hyperparameter found via cross-validation.

Ridge does not perform feature selection.

Lasso Regression (L1)

Prevents overfitting AND performs feature selection by shrinking some coefficients to exactly zero.

Cost = (ŷ − y)² + λ|slope|

Performs feature selection Ridge does not

Assumptions for Best Results

1
Normal distribution — model trains well with Gaussian data
2
Standardization — scale data using Z-score (μ=0, σ=1)
3
Linearity — relationship between features and target is linear
4
No multicollinearity — predictors should not be highly correlated
